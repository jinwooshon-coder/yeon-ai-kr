# 연소사(이진우 선생님)의 최종 수학 프로그램 비전
## — 처음부터 끝까지, 하나도 빠짐없이 —

> 작성일: 2026년 5월  
> 이 문서는 연소사님이 여러 세션에 걸쳐 말씀하신 모든 내용과 수풀AI 실제 코드 분석을 종합한 비전 문서입니다.

---

## 0. 이 프로그램이 존재하는 이유

연소사님이 만들고자 하는 수학 프로그램은 단순한 문제풀이 앱이 아닙니다.

**"현재 학생이 좋아하는 방식을 점차 알아나가야 하고, 점점 더 좋은 방향으로 이끌 수 있어야 해."**

학생마다 이미 굳어진 학습 패턴이 있습니다. 그 패턴을 무시하고 억지로 좋은 방법을 주입하는 것이 아니라, 먼저 학생이 편한 방식으로 설명하고, 그 다음 더 나은 방향을 부드럽게 제안합니다. 데이터가 쌓일수록 프로그램은 학생을 더 깊이 이해하고, 맞춤 서비스는 점점 정교해집니다.

이것이 수풀AI가 단순한 AI 풀이 도구가 아닌 이유입니다. **학생을 알아가는 시스템**입니다.

## 1. 전체 생태계 구조

이 수학 프로그램은 독립된 앱이 아니라 연소사님의 교육 생태계 안에 자리잡습니다.

```
연소사님 교육 생태계
├── 영풀 (영어 학습 프로그램)      ← 수학과 별개, 병렬 운영
├── 수풀AI (수학 학습 프로그램)    ← 이 문서의 핵심
│   ├── 학생용 인터페이스 (su-pul.html → Y-Life 플랫폼으로 발전)
│   ├── 선생님용 도구 (연쌤수첩)
│   └── 관리자 대시보드 (admin.html)
├── 연플래너 (자기주도 학습 플래너)
├── 스무고개 상담카드 (신입 상담)
└── 연학원관리 admin (대시보드)
```

영풀과 수풀AI는 **별도의 독립 프로그램**으로 운영됩니다.

## 2. 수풀AI — 현재 실제 상태 (코드 직접 확인)

### 2-1. 지금 실제로 있는 것 (su-pul.html 코드 기준)

수풀AI(su-pul.html)는 이미 운영 중이며, 실제로 구현된 기능은 다음과 같습니다:

**탭 구조 (현재 5개 탭)**
- `문제풀이` — 시험지 사진 촬영 → OCR 자동 인식 → AI 풀이 요청
- `연습` — 연습문제(학년/단원별) + 자가테스트(타이머 포함)
- `수업` — 수업 노트, 숙제 관리, 수업 달력
- `공식` — 공식 학습 (단원별, 레벨별)
- `내 학습` — 학습 기록 타임라인, 복습 스케줄

**실제 구현된 기술 요소**
- **OCR (Tesseract.js)**: 학생이 문제 사진을 찍으면 자동으로 텍스트 인식
- **KaTeX**: 수식을 수학 교재처럼 깔끔하게 렌더링
- **화이트보드**: 문제 위에 직접 그리며 풀기 가능
- **학생 ID 시스템**: `yeon_session` / `yeon_auto_login`으로 학생 식별 이미 작동 중
- **자가테스트**: 타이머 포함, 학년/단원 선택 후 문제 풀기
- **복습 시스템 구조**: 플래너 연동 형태로 코드에 구현됨
- **프린트 기능**: A4 용지 출력 (1단/2단 레이아웃 선택)
- **카카오 인앱 브라우저 감지**: 외부 브라우저로 자동 리다이렉트
- **다크모드**: CSS 변수로 이미 지원
- **개선 요청 게시판**: 학생이 불편한 점을 직접 제보하는 탭
- **에러 모니터링**: 전역 에러 로그 시스템 내장

**현재 데이터 저장 방식**
- 모든 학습 데이터는 **기기 내 localStorage에 저장** 중
- "본부 서버 업로드 대기" 플래그가 코드에 있으나 **실제 서버 전송 코드는 아직 없음**
- 학생 ID는 있지만 서버로 보내는 파이프라인이 미완성

### 2-2. 아직 없는 것 (다음 단계에서 만들어야 할 것)

- **3가지 풀이 선택 UI**: "3가지 풀이법을 드립니다"라는 문구는 있지만, 실제로 학생이 3가지 중 하나를 선택하고 그 선택이 기록되는 화면이 없음
- **서버 실제 전송**: 학생 데이터가 본사 서버로 올라가는 실제 코드
- **학생 프로파일 누적**: 선택 데이터가 쌓여서 개인 맞춤 서비스로 이어지는 로직
- **맞춤형 풀이 제공**: 학생 패턴에 따라 다른 순서로 설명하는 기능

### 2-3. 문제은행 (구축 중)

- **대상**: 중1 ~ 고1
- **난이도**: Lv.1 (기초) ~ Lv.5 (심화)
- **학기별 분류**: 1학기 중간고사 기준 단원 구성
- **자동 유사문제 생성**: 학생이 올린 문제를 AI가 분석 → 비슷한 유형 자동 생성
- **60문제 이상** 이미 구축, 계속 확장 중
- 저장 위치: `G:\내 드라이브\연-학과목데이터뱅크\학년\수학\문제은행\`

### 2-4. 순환학습 시스템 (설계 완료 — 우혁이 실험으로 검증)

```
문제 올리기 → AI 풀이 분석 → 피드백 → 재출제
      ↑                                        ↓
      └────────────────────────────────────────┘
```

1. **올리기**: 학생이 모르는 문제 사진을 찍어 올림 (OCR 자동 인식)
2. **분석**: AI가 풀이 과정을 보고 어디서 막혔는지 파악
3. **피드백**: 최적 풀이 방법 제시 + 오류 설명
4. **재출제**: 같은 개념의 다른 문제로 다시 도전

이 사이클이 반복되면서 실력이 쌓이고 데이터가 누적됩니다.

## 3. 핵심 비전 — 학생을 알아가는 3단계

### 3-1. 1단계: 데이터 수집 (3풀이 선택 시스템)

학생이 문제를 올리면 **3가지 다른 접근법의 풀이**를 보여줍니다.

```
[문제: 이차방정식 x²-5x+6=0]

풀이 ① 인수분해법     ← 이걸 선택했어요 ✓
풀이 ② 근의 공식
풀이 ③ 완전제곱식

"이 풀이 방법이 제일 이해가 됐어요" → 선택 → 서버에 기록
```

이 선택 하나하나가 **학생의 사고방식을 읽는 데이터**가 됩니다.

### 3-2. 2단계: 패턴 인식 (학생 프로파일 형성)

10~20번의 선택이 쌓이면 패턴이 보이기 시작합니다.

```json
{
  "student_id": "김우혁",
  "preferred_explanation_style": "시각적/단계별",
  "weak_concepts": ["인수분해", "이차방정식"],
  "strong_concepts": ["일차함수"],
  "solution_preferences": {
    "풀이①_선택횟수": 12,
    "풀이②_선택횟수": 3,
    "풀이③_선택횟수": 8
  },
  "learning_pattern": "직관적 접근 선호",
  "improvement_trajectory": [...],
  "review_schedule": {...}
}
```

### 3-3. 3단계: 맞춤 서비스 (2단계 설명)

충분한 데이터가 쌓이면, 이제 수풀AI가 먼저 나섭니다.

```
1단계: 학생이 잘 이해하는 방식으로 먼저 설명
    ↓
2단계: "사실 이렇게 하면 더 빠르게 풀 수 있어" → 더 나은 방법 제안
    ↓
학생 반응 기록 (수용 or 거부) → 프로파일 업데이트
```

이것이 **"현재 학생이 좋아하는 방식을 알아가면서, 점점 더 좋은 방향으로 이끄는"** 핵심 흐름입니다.
갑자기 바꾸는 게 아니라, 학생의 방식을 인정하면서 조금씩 더 나은 곳으로 데려갑니다.

## 4. 풀이법 DB — 2트랙 구조

**중요 원칙**: 우리는 장대경 선생님의 추종자가 아닙니다.

모든 수학 문제에는 두 가지 트랙의 풀이법이 준비됩니다:

```
[트랙 A] 최적 일반 풀이법
  → 수학적으로 가장 정확하고 범용적인 방법
  → 어떤 상황에도 쓸 수 있는 방법

[트랙 B] 장대경식 풀이법
  → 장대경 선생님만의 직관적/빠른 접근법
  → 특정 유형에서 매우 강력한 방법
```

미래에 더 좋은 풀이법이 발견되면 트랙 C, D로 계속 추가됩니다. 지식 그래프는 고정이 아니라 계속 성장합니다.

### Obsidian 지식 그래프 구조

```
레이어 1: 개념별 풀이법 노드
├── [이차방정식]
│   ├── 최적 풀이법 (인수분해 우선)
│   ├── 장대경식 풀이법 (십자곱 직관법)
│   └── (미래에 추가될 풀이법들)
├── [인수분해]
│   └── ...

레이어 2: 학생 패턴 노드
├── [김우혁] — 직관적, 그림 선호
├── [이민준] — 계산 과정 선호
└── ...

레이어 3: 연결 노드 (살아있는 부분)
├── "우혁이에게는 장대경식이 더 잘 맞음"
├── "민준이는 최적 풀이법이 더 효과적"
└── (데이터가 쌓일수록 자동으로 새 연결 생성)
```

## 5. 3대 데이터베이스 구조

### DB 1: 문제 DB
- 중1~고1 전 단원 수학 문제
- 유형, 단원, 난이도(Lv.1~5), 출처
- 각 문제에 연결된 개념 태그
- 자동 유사문제 생성

### DB 2: 풀이법 DB
- Obsidian 지식 그래프로 관리
- 최적 풀이법 + 장대경식 풀이법 (+ 미래 추가분)
- 각 개념별로 어떤 풀이법이 어떤 학생 유형에 잘 맞는지 연결

### DB 3: 학생 DB
- 학생별 개인 프로파일 (student_profile.json)
- 선호 풀이 방식, 취약 개념, 강점 개념
- 학습 이력, 복습 스케줄
- 맞춤 문제 제시를 위한 기반

## 6. 콘텐츠 수집 파이프라인

### 6-1. 3가지 입력 소스

```
[소스 1] YouTube 강의 영상 (장대경 선생님 등)
[소스 2] Google Drive 저장 영상 (대구학원 폴더 등)
[소스 3] GoodNotes iPad 강의 → PDF 직접 내보내기
```

**핵심 원칙**: URL 하나만 입력하면 나머지는 자동. 수동 단계 없음.

### 6-2. 자동화 처리 엔진

**Twelve Labs** (핵심 엔진)
- 한국계 창업자의 비디오 AI
- 영상에서 칠판 손글씨(시각)와 음성(청각)을 동시에 분석
- 기존의 "프레임 추출 → OCR → 별도 음성인식" 3단계를 1단계로 통합

**Vrew** (음성 → 타임스탬프 추출)
- 강의 음성을 타임스탬프 포함 텍스트로 변환
- 수학 선생님은 항상 "15번 문제 보겠습니다" 등을 구두로 말함
- "문제", "번", "보겠습니다" 키워드 자동 감지 → 문제 시작 타임스탬프 자동 추출
- 수동으로 영상을 보며 타임스탬프를 기록하는 작업 완전 제거

**LaTeX / Typst 변환**
- Twelve Labs가 인식한 수식 → LaTeX 변환
- 최종 교재 PDF 출력에는 Typst 사용 (현대적 LaTeX 대안)

### 6-3. GoodNotes 특별 처리 경로

아이패드에서 PDF 위에 GoodNotes로 필기한 강의:
- 영상 분석 파이프라인 없이 **PDF 직접 내보내기**로 완성
- 가장 빠르고 깔끔한 콘텐츠 수집 경로

### 6-4. 분석 대상 콘텐츠

**장대경 선생님 강의**
단순히 문제와 답이 아니라 **방법론 자체**를 분석:
- 어떤 순서로 접근하는가
- 어떤 개념을 먼저 적용하는가
- 어떤 경우에 어떤 방법을 선택하는가
- 그만의 독특한 시각화/직관법은 무엇인가

이미 장대경 선생님 학생용 PWA 앱도 제작 완료됨.

**깨봉수학 (참고 콘텐츠)**
- 344개 유튜브 영상 이미 수집 완료
- 분석 대시보드 제작됨, 자동 수집 스케줄 설정됨
- 직관적 수학 설명 방식을 참고 콘텐츠로 활용

## 7. 5.2.5.5 복습 체계 연동

연소사님의 핵심 교육 철학을 수풀AI에 자동 연동:

- **5분 후**: 문제 풀고 나서 5분 뒤 핵심 개념 요약 재확인
- **2시간 후**: 비슷한 유형의 문제 하나 더 제시
- **5시간 후**: 같은 문제를 다시 풀어보기
- **5일 후**: 완전히 다른 맥락에서 같은 개념 적용 문제 제시

정해진 시간이 되면 자동으로 알림과 문제가 발송됩니다. 복습 스케줄은 학생 DB에 기록됩니다.
(현재 수풀AI에 복습 구조 코드가 있으나, 서버 연동 없이 기기 내에서만 동작 중)

## 8. Y-Life 플랫폼 (학생 향 배포)

학생들이 콘텐츠를 보는 화면:

- **타임스탬프 연동 버튼**: "이 부분 영상으로 보기" 누르면 해당 강의의 정확한 시점으로 바로 이동
- 문제 → 풀이 설명 → 영상으로 바로 가기가 한 화면에서 이루어짐
- 모든 기기 지원 (갤럭시폰, 아이폰, 아이패드, 노트북, PC)

## 9. 연쌤수첩 (선생님용 도구)

연소사님이 수업을 준비하고 관리하는 선생님 전용 앱:

- 수업 준비 사항 기록
- 학생별 진도 및 특이사항 관리
- 강의 콘텐츠 연결 및 관리
- 카카오톡으로 학부모에게 수업 내용 링크 전송
- 수업 중 화면 잠금 "수업 모드"

    ↓

    ↓

    ↓

    ↓
선택 데이터 → 서버 전송 → 학생 프로파일 누적
    ↓

    ↓
[2단계 설명]

2단계: 더 나은 방법 부드럽게 제안
    ↓
[5.2.5.5 복습 스케줄]
자동 알림 + 복습 문제 발송
    ↓

## 12. 다음 핵심 과제 — 무엇을 먼저 만들어야 하는가

비전 전체에서 가장 중요한 **빠진 고리**는 하나입니다:

> **학생이 3가지 풀이를 보고 선택하면 → 그 데이터가 서버에 쌓이는 것**

이것 하나가 완성되면 나머지가 모두 연결됩니다. 서버에 데이터가 쌓여야 학생 프로파일이 형성되고, 프로파일이 있어야 맞춤 서비스가 가능합니다.

수풀AI에 이미 잘 만들어진 기반(OCR, 화이트보드, 복습 구조, 학생 ID)이 있습니다. 그 위에 3풀이 선택 UI + 서버 전송 한 세트를 얹으면, 전체 비전이 돌아가기 시작합니다.

## 13. 이 프로그램이 완성되면

민준이가 인수분해 문제를 사진 찍어 올립니다. 수풀AI가 세 가지 방식의 풀이를 보여주고, 민준이는 단계별 계산 방식을 선택합니다. 그 선택이 서버에 기록됩니다.

열 번째 문제쯤 됐을 때, 수풀AI는 민준이가 단계별 계산을 좋아한다는 것을 알고 있습니다. 다음 문제가 나오면 자동으로 그 방식으로 먼저 설명합니다. 그런 다음 "사실 이렇게 짧게도 풀 수 있어" 하며 장대경식 빠른 풀이를 제안합니다.

민준이가 흥미를 보이면 기록됩니다. 거부하면 그것도 기록됩니다.

5시간 뒤, 수풀AI가 자동으로 복습 문제를 보냅니다. 3개월이 지나면 민준이의 프로파일은 정교해져 있고, 수풀AI는 민준이에게 가장 잘 맞는 방식으로 정확히 안내합니다.

이것이 연소사님이 꿈꾸는 수학 프로그램입니다.
